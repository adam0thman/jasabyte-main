SAP Self Service Password Reset (using NodeJS)

This package arise from a typical pain point from IT Service Management where SAP end-users keeps on issuing P3 password reset tickets that may somewhat take a lot of time in resolution. A proactive approach to this would be to have Single Sign On, while others keep to the solution of Self Service Password Reset. Jasabyte provides both.

Package Details
Engagement Model Subscription-based Pricing determined via a yearly subscription per SAP user ID.
Price MYR 16.00 Per user Per Year
Implementation Cost MYR 0.00 Inclusive of any implementation cost.
Implementation Time 2 - 5 business days Please see the Requirement section below for further details.
Requirement:

To ensure the efficient implementaation of this solution, we would need to lay a few pre-conditions before any engagement starts:

  1. An Intel-based server environment for Self Service Portal hosting. Other processoir architecture are not supported.
    1. Min 4GB memory
    2. Min 1 CPU Core
    3. Min 20GB Disk Space
  2. SSL Certificate (and its relevant certificate chain) in PEM format. This is applicable for requirement of serving the web-based Self Service Portal via HTTPS.
  3. Administrators / technical expert for:
    1. DNS - its typical for the portal to be accessed via a internal corporate (sub)domain
    2. Networking - ensuring optimize routing for internal network to the portal service
    3. Firewall - for allowing end-user access to the portal
  4. Key business users for User Acceptance Test
Frequently Asked Questions:
My corporate practice emphasizes on a 3-tier Dev, QA and PRD landscape. Does this mean I am charged tripled?

Fret not. The package will cover setup for at most 3 environment, within the same landscape. You will be to comply with your organization corporate practices and answer to auditors handsomely.

I have multiple envrioonements, wth different user IDs. How do we calculate the user total then?

The user subscription are calculated from Production environment only. It is a Named User based, just like SAP. This includes Disaster Reocvery environment as well.

Does this applicable to only SAP ECC or S/4HANA application? What about BW, et cetera?

Our solutiion integrates and works directly with SAP applications that rides on NetWeaver ABAP stack as well as S/4HANA ABAP Platform. Other then ECC and S/4HANA, we can also support other ABAP-based applications such as SAP BW, Front-End Fiori Server, EWM servers, et cetera.
